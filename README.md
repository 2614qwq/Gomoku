
# 五子棋 —— 棋谱驱动·全新技能·多智能体协作版

基于 tkinter 的五子棋游戏，以 **CSV 棋谱数据为底层模型**（int 编号体系），搭载 **10 种全新主动技能**（每局限 1 次、必成功），集成**传统算法引擎**（棋型检测 + Minimax/Alpha-Beta 搜索）与基于 **LangGraph** 的多智能体 LLM 系统。支持双人对战、人机对战、AI vs AI 观战模式。

---

## 快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置 API Key（AI 相关功能需要）

## 底层模型：CSV 棋谱编号体系

本版本的底层 Board 模型基于 `chess_records.csv`（94 条专业棋谱）重构：

- **int 网格**：`_grid[y][x]` 存储 `int`——`0`=空位，奇数 (1,3,5...) = 黑棋，偶数 (2,4,6...) = 白棋，数字大小 = 落子顺序
- **双轨输出**：Agent 同时接收两种棋盘表示——
  - **X/O 可视化棋盘**（`board_to_text()`）：便于 LLM 识别活二/活三/活四的几何形态
  - **落子序列**（`board_to_move_sequence()`）：按编号排序的完整走法时间线，奇数=黑、偶数=白
- **棋谱重放**：int 网格天然支持按序号递增依次落子，可完整重放 CSV 中任意一局

## 技能系统（10 种主动技能）

每局双方各随机分配 **1 个技能**（确保双方不同）。**全部为主动技能**，每局限 1 次，必成功。

### 回合内技能时序

```
你的回合开始
    │
    ├── ① [可选] 使用主动技能（每局限1次）
    │         └── 技能效果立即生效（增/删/移/封）
    │
    ├── ② 正常落子（必须）
    │
    └── ③ 回合结束 → 切换至对手回合
```

**关键约束**：技能只能在落子前发动。先落子则回合立即切换，技能窗口关闭。技能与落子应互相配合——技能为落子创造更优条件。

### 进攻档（3 个）

| # | 技能 | 可用回合 | 效果 |
|---|------|---------|------|
| 1 | **追加落子** | ≥5 | 在当前回合额外在任意空位放置 1 子，之后仍可正常落子 |
| 7 | **复制己子** | ≥3 | 选择己方 1 子，在其相邻 4 格空位中生成 1 颗同色子 |
| 9 | **连打两子** | ≥10 | 本回合连续落下 2 子（而非正常的 1 子），可一次性构造必胜定式 |

### 控制档（4 个）

| # | 技能 | 可用回合 | 效果 |
|---|------|---------|------|
| 2 | **移除敌子** | ≥3 | 选择并移除对手任意 1 颗棋子 |
| 3 | **转化棋子** | ≥8 | 选择对手 1 颗棋子，将其颜色变为己方 |
| 4 | **封禁空位** | ≥3 | 标记 1 个空位为"封锁"，对手下 1 回合不能落此位 |
| 10 | **强制跳过** | ≥8 | 对手跳过下个回合（你连续走两轮） |

### 技巧档（3 个）

| # | 技能 | 可用回合 | 效果 |
|---|------|---------|------|
| 5 | **移位己子** | ≥3 | 选择己方 1 子，移动到相邻 8 格内的任意空位 |
| 6 | **交换棋子** | ≥8 | 选择己方 1 子与对手 1 子，交换两者位置 |
| 8 | **回溯落子** | ≥5 | 撤销自己上一步落子，恢复该位置为空，重新选择落子位置 |

### 与旧版技能对比

| 变化 | 说明 |
|------|------|
| 全部主动化 | 旧版 7 被动 + 3 主动 → 新版 10 主动，Agent 完全可控 |
| 去除概率 | 旧版"10% 概率""40% 概率" → 新版全部必成功 |
| 去除"技能子"概念 | 旧版五雷阵/绝户阵依赖"技能生成子"标签 → 新版无此概念 |
| 算法归位 | 旧版归元阵/克敌先机的"连 4 补防" → 改由算法预检自动处理 |

## 多智能体 AI 架构

### 整体架构

```
CSV棋谱 (chess_records.csv)
       │
       ▼
GameController ──→ 算法预检（必胜/必堵/双重威胁）

       │                │ 无强制落子 ↓
       │         GameInfoExtractor ──→ MultiAgentOrchestrator (LangGraph)
       │                │                      │
       │         ┌──────┴──────┐               │
       │         │             │               │
       │   board_to_text  board_to_move        │
       │   (X/O 可视化)   _sequence            │
       │   (棋型识别)     (走法时间线)           │
       │         │             │               │
       │         └──────┬──────┘               │
       │              game_report              │
       │                                      │
       │         ┌────────────────────────────┼──────────────────────────┐
       │         │                            │                          │
       │   ┌─────▼─────┐              ┌──────▼──────┐           ┌──────▼──────┐
       │   │  战术官    │              │   防守官     │           │ 技能使用官   │

       │   │  进攻分析   │              │   防守分析    │           │ 技能裁决    │
       │   └─────┬─────┘              └──────┬──────┘           └──────┬──────┘
       │         │                            │                          │
       │         └────────────────────────────┼──────────────────────────┘

       │                               ┌──────▼──────┐
       │                               │  总策划官    │

### 决策流程

```
analyze() 入口
  │
  ├─ 算法预检（_algorithm_precheck）
  │    ├─ 己方必胜点 → 直接落子，返回（0 次 LLM）
  │    ├─ 对手必胜点 → 强制封堵，返回（0 次 LLM）
  │    └─ 己方双重威胁 → 直接进攻，返回（0 次 LLM）
  │
  ├─ 快速开局通道（turn 1-4）
  │    ├─ turn=1 → 硬编码天元 (7,7)
  │    └─ turn=2-4 → 单次 LLM 快速决策
  │
  └─ 完整多智能体流水线（turn ≥ 5）
       │
       START → phase_check
                └─ Send("tactical") ║ Send("defense")   (LangGraph 并行)
                              ↓                    ↓
                         post_analysis ← post_analysis   (join 屏障)
                                    ↓
                        ┌─ 共识 → consensus → END
                        └─ skill_officer → chief → END
       │
       └─ 重试耗尽 → 算法搜索兜底（Minimax depth=2）
```

### LangGraph 图结构

```
START → phase_check → [Send(tactical) || Send(defense)]
                           ↓                    ↓
                      post_analysis ←── post_analysis
                           ↓
                 ┌─ 共识(同坐标+置信度>0.8) → consensus → END
                 └─ skill_officer → chief → END
```

### 四个智能体

| 角色 | 模型 | 职责 |
|------|------|------|
| **战术官** TacticalAnalyst | qwen-turbo | 寻找进攻机会，结合己方技能优化策略；通过落子序列理解走法时间线 |
| **防守官** DefenseSpecialist | qwen-turbo | 识别对手威胁，预判敌方技能可能造成的破坏，调整防守强度 |
| **技能使用官** SkillOfficer | qwen-plus | 根据 game_report 中标注的"技能使用窗口"信息，裁决是否在当前回合激活技能 |
| **总策划官** ChiefStrategist | qwen-plus | 汇总所有意见（含技能裁决），确保落子与技能效果形成配合，做出最终裁决 |

### 性能优化

| 优化项 | 说明 |
|--------|------|
| **算法预检短路** | 必胜/必堵/双重威胁直接由算法处理，0 次 LLM 调用，响应 < 1ms |
| **LangGraph Send fan-out** | tactical 和 defense 通过 LangGraph 原生 `Send` API 并行执行 |
| **共识短路** | tactical 和 defense 建议相同落子且置信度 > 0.8 时，跳过 chief |
| **双轨棋盘输出** | X/O 可视化棋盘 + 落子序列，Agent 同时拥有空间视图和时间视图 |
| **技能使用窗口标注** | game_report 显式标注每种技能的"最佳使用时机"，降低 Agent 裁决难度 |
| **后台线程** | LLM 调用在后台线程执行，主线程不阻塞，UI 全程流畅 |
| **算法搜索兜底** | LLM 重试耗尽后使用 Minimax + Alpha-Beta 搜索，替代随机落子 |
| **滑动窗口记忆** | 短时记忆（2 步）+ 关键事件（2 条），控制在 500 字符内 |
| **候选剪枝** | 搜索仅限已有棋子周围 2 格空位，大幅缩小分支因子 |

## 设计原则

- **CSV 棋谱驱动**：底层 Board 模型基于 `chess_records.csv` 的 int 编号体系（奇数=黑，偶数=白），与专业棋谱数据对齐
- **双轨 Agent 可见**：Agent 同时接收 X/O 可视化棋盘（空间形态）和落子序列（时间线），两者结合决策
- **技能必成功、完全可控**：全部为主动技能，Agent 只需判断"是否值得使用"，不存在概率命中问题
- **OOP 单一职责**：一个文件一个核心类，封装行为，接口交互
- **高内聚低耦合**：Agent 之间互不知晓，仅通过 LangGraph State 传递信息
- **LangGraph 原生**：并行、路由、状态管理全部由 LangGraph 框架处理
- **线程安全**：LLM 调用在后台线程执行，UI 更新全部在主线程回调中完成
- **渐进式 AI**：简单局用引擎，复杂局用多智能体，共识时自动短路
