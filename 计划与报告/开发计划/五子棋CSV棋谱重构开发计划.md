# 五子棋CSV棋谱重构开发计划

> 基于 chess_records.csv 棋谱数据，重构游戏基础逻辑与技能系统。
> 状态：设计稿 | 日期：2026-05-11

---

## 一、项目目标

将当前"万宁五子棋"（超自然招式 + X/O字符棋盘）重构为：

- **底层模型**：基于 `chess_records.csv` 的编号体系（奇数=黑，偶数=白）
- **技能系统**：10个主动技能，每局限1次，必成功，条件清晰
- **智能体提示词**：精准描述每个技能的使用时机，Agent能明确判断

---

## 二、CSV棋谱数据格式分析

### 2.1 数据格式

```
board_name | source_image | md5 | grid (15×15 JSON数组)
```

- 共94条棋谱记录
- 每条记录是一个15×15矩阵
- `0` = 空位
- 奇数 (1,3,5,...) = 黑棋落子
- 偶数 (2,4,6,...) = 白棋落子
- 数字大小 = 落子顺序（1最先，数字越大越靠后）

### 2.2 棋谱特性

| 特性 | 说明 |
|------|------|
| 棋盘大小 | 15×15（与当前系统一致） |
| 走法数 | 最少7手，最多82手 |
| 开局占位 | 几乎全部从天元(7,7)附近开始 |
| 数字间距 | 不连续跳跃说明存在被吃/提子（连珠规则中罕见，暗示部分棋谱含禁手判负后的空位） |

## 三、游戏基础逻辑改造

### 3.1 Board模型重构

**当前**：`_grid[y][x]` 存储 `' '` / `'X'` / `'O'`（字符型）

**目标**：`_grid[y][x]` 存储 `int`（0=空，奇数=黑，偶数=白，数字=落子序号）

```python
# 新增属性
class Board:
    _grid: list[list[int]]       # 0=空, 1,2,3...=序号
    _move_count: int              # 当前总落子数（=最大序号）
    
    def get_color(self, x, y) -> str:    # 1,3,5→'X', 2,4,6→'O', 0→' '
    def get_move_number(self, x, y) -> int:  # 返回落子序号
    def place(self, x, y, color) -> int:     # 返回新序号
    def replay_from_record(self, grid_data): # 从CSV加载一局
```

**面向Agent的双轨输出**：`board_to_text()` 保持X/O/*可视化棋盘（LLM易于识别棋型），同时新增 `board_to_move_sequence()` 输出完整落子序列（编号+坐标），两者共同注入 `game_report`。Agent同时看到"棋盘长什么样"和"每一步是谁走的"，奇数=黑、偶数=白，与CSV棋谱完全对应。

### 3.2 board_codec 双轨输出（Agent可见层）

Agent 通过 `game_report` 同时接收两种表示：

```
【棋盘】（X/O/* 可视化，便于棋型识别）
 0 1 2 3 4 5 6 7 ...
0 . . . . . . . . ...
1 . . . . . X . . ...
2 . . . O . X . . ...
...

【落子序列】（CSV编号格式，便于理解走法顺序）
黑棋(奇数): 1.(7,7) 3.(6,8) 5.(8,8) 7.(5,9) ...
白棋(偶数): 2.(8,6) 4.(7,8) 6.(8,9) 8.(7,6) ...
```

**设计理由**：
- X/O棋盘：Agent一眼识别活二/活三/活四的几何形态，与现有prompt兼容
- 落子序列：Agent理解走法的时间顺序，奇数=黑、偶数=白，与CSV棋谱一致
- RAG检索：用int matrix（0/1/2/3...）匹配CSV棋谱，找相似局面

## 四、技能系统重设计（10主动技能）

### 4.0 设计原则

```
┌──────────────────────────────────────────────────┐
│  原则1: 全部为"主动技能"，每局限1次，必成功       │
│  原则2: 技能在"落子前"使用（你的回合内）         │
│  原则3: game_report 显式标注技能的"最佳使用窗口" │
│  原则4: 技能值体现在"破坏对手结构"或"加速己方进攻"│
│  原则5: 落子后回合切换，技能不可再发动            │
└──────────────────────────────────────────────────┘
```

### 4.1 游戏回合内的技能时序

```
你的回合开始
    │
    ├── ① [可选] 使用主动技能（每局限1次）
    │         └── 技能效果立即生效（增/删/移/封）
    │
    ├── ② 正常落子（必须）
    │
    └── ③ 回合结束 → 切换至对手回合
```

**关键约束**：技能只能在落子前发动。如果先落子，回合立即切换，技能窗口关闭。

这个约束意味着：
- Agent需要**先决策是否用技能，再决策落子位置**
- 技能使用后**可以改变局面**，从而影响后续落子选择
- 技能使用与落子应该**互相配合**：技能为落子创造更优条件

### 4.2 10技能完整设计

---

#### 技能1：追加落子（Extra Stone）

| 属性 | 内容 |
|------|------|
| **效果** | 在当前回合额外在任意空位放置1子（之后你仍可正常落子） |
| **可用回合** | ≥ 第5回合 |
| **使用时机** | 局面存在"只差1子"即可形成活三/活四/双重威胁 |
| **game_report标注** | `技能-追加落子: 可用, 第5回合起有效` |

**Agent判断逻辑**：
```
IF 场上存在空位E，在此落子使己方从活二→活三 或 冲三→活四
THEN 使用追加落子在E → 正常落子选另一处形成双杀
```

**示例**：己方有活二 a-b，空位c可使其变活三。用追加落子在c→正常落子在d形成另一个活三→对手首尾难顾。

---

#### 技能2：移除敌子（Remove Enemy）

| 属性 | 内容 |
|------|------|
| **效果** | 选择并移除对手任意1颗棋子 |
| **可用回合** | ≥ 第3回合 |
| **使用时机** | 对手存在活三/冲四，且移除某一颗能破坏该连线 |
| **game_report标注** | `技能-移除敌子: 可用, 对手活三棋子在(3,7)(4,7)(5,7), 移除任一端可破坏` |

**Agent判断逻辑**：
```
IF 对手存在活三或冲四，算法标注了该连线的所有棋子位置
THEN 选择连线端点的棋子移除 → 正常落子加固己方
```

---

#### 技能3：转化棋子（Convert）

| 属性 | 内容 |
|------|------|
| **效果** | 选择对手1颗棋子，将其颜色变为己方（X↔O） |
| **可用回合** | ≥ 第8回合 |
| **使用时机** | 对手关键棋子处于己方潜在连珠路线上 |
| **game_report标注** | `技能-转化棋子: 可用(≥8回合), 对手棋子分布: (x1,y1)(x2,y2)...` |

**Agent判断逻辑**：
```
IF 己方有活二或活三，但其延伸方向上被对手棋子占据关键位
THEN 转化该对手棋子为己方 → 一举完成活三/活四
```

---

#### 技能4：封禁空位（Block Position）

| 属性 | 内容 |
|------|------|
| **效果** | 标记1个空位为"封锁"，对手下1回合不能落此位 |
| **可用回合** | ≥ 第3回合 |
| **使用时机** | 对手存在活三/冲四，且堵住某一端可使其失效1回合 |
| **game_report标注** | `技能-封禁空位: 可用, 对手威胁延伸空位: (x1,y1)(x2,y2)` |

**Agent判断逻辑**：
```
IF 对手存在活三（算法标注了两端空位）
THEN 封禁其中一端 → 对手活三暂时无法扩展 → 为己方争取1回合
```

---

#### 技能5：移位己子（Shift Own）

| 属性 | 内容 |
|------|------|
| **效果** | 选择己方1子，将其移动到相邻8格内的任意空位 |
| **可用回合** | ≥ 第3回合 |
| **使用时机** | 己方某棋子在孤位（不在任何连珠线上），移入可形成连接 |
| **game_report标注** | `技能-移位己子: 可用, 己方孤立子: (x1,y1), 可移至: [(a,b),(c,d)...]` |

**Agent判断逻辑**：
```
IF 己方存在孤立棋子（不在任何≥2连珠中）
   且其相邻有空位可使该子加入已有活二/活三
THEN 移位该子 → 增强连线
```

---

#### 技能6：交换棋子（Swap）

| 属性 | 内容 |
|------|------|
| **效果** | 选择己方1子与对手1子，交换两者位置 |
| **可用回合** | ≥ 第8回合 |
| **使用时机** | 对手关键棋子与己方闲置棋子互换后，同时破坏对手+增强己方 |
| **game_report标注** | `技能-交换棋子: 可用(≥8回合)` |

**Agent判断逻辑**：
```
IF 对手活三线上棋子P，己方有孤立子Q
   且交换后P变己方破坏对手活三，Q变对手棋子但无威胁
THEN 交换P与Q
```

---

#### 技能7：复制己子（Clone）

| 属性 | 内容 |
|------|------|
| **效果** | 选择己方1子，在其相邻4格（上下左右）空位中生成1颗同色子 |
| **可用回合** | ≥ 第3回合 |
| **使用时机** | 己方核心棋子需要"分叉"进攻 |
| **game_report标注** | `技能-复制己子: 可用, 己方棋子相邻空位候选: [(a,b),(c,d)...]` |

**Agent判断逻辑**：
```
IF 己方某棋子处于活二上，其相邻空位落子可形成活三
THEN 复制该子到该相邻空位 → 活二升级为活三
```

---

#### 技能8：回溯落子（Retract）

| 属性 | 内容 |
|------|------|
| **效果** | 撤销自己上一步落子，恢复该位置为空，重新选择落子位置 |
| **可用回合** | ≥ 第5回合，且上一步必须是自己的回合 |
| **使用时机** | 上一步落子后算法检测到对手形成必胜威胁，且该威胁与你刚落的子直接相关 |
| **game_report标注** | `技能-回溯落子: 可用, 上一步(7,7)导致对手可形成活四` |

**Agent判断逻辑**：
```
IF 上一步是自己的，且该位置与对手当前活四/活三有直接关系
THEN 回溯撤销 → 重新选一个更安全的位置落子
```

---

#### 技能9：连打两子（Double Move）

| 属性 | 内容 |
|------|------|
| **效果** | 本回合连续落下2子（而不是正常的1子） |
| **可用回合** | ≥ 第10回合 |
| **使用时机** | 可以一次性构造必胜定式（活四/四三杀/双活三） |
| **game_report标注** | `技能-连打两子: 可用(≥10回合)` |

**Agent判断逻辑**：
```
IF 场上存在空位A和B，同时落下A+B可形成活四或四三杀
THEN 激活连打两子 → 落子A+B → 直接必胜
```

**注意**：这是最强技能，限制在第10回合后才可用，确保中盘前不会过早结束。

---

#### 技能10：强制跳过（Skip Opponent）

| 属性 | 内容 |
|------|------|
| **效果** | 对手跳过下个回合（你连续走两轮） |
| **可用回合** | ≥ 第8回合 |
| **使用时机** | 你需要连续两手才能构成威胁，但中间不能让对手有机会堵 |
| **game_report标注** | `技能-强制跳过: 可用(≥8回合)` |

**Agent判断逻辑**：
```
IF 己方存在"只需连续两手"即可构造必胜的路线
   但若正常轮替对手会在中间堵住
THEN 跳过对手回合 → 连续下两手 → 必胜
```

### 4.3 技能对比总览

| # | 技能名 | 可用回合 | 效果 | 最佳使用窗口（Agent判断依据） |
|---|--------|---------|------|------|
| 1 | 追加落子 | ≥5 | 额外放1子 | 存在"只差1子成双杀"的空位 |
| 2 | 移除敌子 | ≥3 | 删除对手1子 | 对手活三/冲四，删端点可破坏 |
| 3 | 转化棋子 | ≥8 | 对手1子变己方 | 对手子占据己方连珠路线关键位 |
| 4 | 封禁空位 | ≥3 | 封锁1空位1回合 | 对手活三延伸空位，封1端争取时间 |
| 5 | 移位己子 | ≥3 | 己方1子移相邻空位 | 己方孤立子可移入连珠线 |
| 6 | 交换棋子 | ≥8 | 己方1子↔对手1子 | 同时破坏对手+增强己方 |
| 7 | 复制己子 | ≥3 | 复制1子到相邻空位 | 核心棋子分叉进攻 |
| 8 | 回溯落子 | ≥5 | 撤回上一步重走 | 上步导致对手形成必胜威胁 |
| 9 | 连打两子 | ≥10 | 本回合落2子 | 可一次构造活四/四三杀 |
| 10 | 强制跳过 | ≥8 | 对手跳过1回合 | 需要连续2手且不能被堵 |

### 4.4 技能分配规则

- 每局双方各**随机分配1个**技能（与当前分配逻辑一致）
- 双方技能**确保不同**
- 10个技能中分为三档：

| 档位 | 技能 | 特点 |
|------|------|------|
| 进攻档 | 追加落子、复制己子、连打两子 | 直接增加己方棋子或落子次数 |
| 控制档 | 移除敌子、转化棋子、封禁空位、强制跳过 | 削弱对手或限制对手行动 |
| 技巧档 | 移位己子、交换棋子、回溯落子 | 调整已落棋子位置 |

## 五、智能体Prompt更新

### 5.1 SkillOfficer 提示词（核心改动）

```
你是五子棋"技能使用官"，负责裁决当前回合是否使用己方主动技能。

【格式说明】
X=黑棋(奇数号) / O=白棋(偶数号)。落子序列段落列出每一步编号→坐标→颜色，
你可据此理解局面是如何一步步发展至此的。

【你的权力】
你拥有1个主动技能（随机分配），每局限使用1次。技能必须在落子前使用；
落子后回合切换，技能窗口关闭。

【技能使用黄金法则】
1. 技能 + 落子 = 一个完整回合。先技能后落子，两者配合。
2. 技能的价值 = 用了比不用"多赚了多少"。无净收益就不使用。
3. 所有技能必成功，不存在概率。你只需要判断"是否值得"。
4. 若技能条件不满足（回合数不够/上步不是自己），activate_skill填null。

【各技能使用时机速查】

▶ 追加落子：算法标注了"差1子成双杀"的空位 → 使用。否则不用。
▶ 移除敌子：对手存在活三或冲四，算法列出了威胁棋子 → 选择端点棋子移除。对手无威胁不用。
▶ 转化棋子：对手棋子恰在己方活二/活三延伸路线上 → 转化该子。否则不用。
▶ 封禁空位：算法标注了对手活三的两端空位 → 封禁一端。否则不用。
▶ 移位己子：己方有孤立子（不在任何连珠线），且移入后可加入连线 → 使用。否则不用。
▶ 交换棋子：对手活三线上棋子 + 己方有闲置子可换 → 使用。≥8回合。
▶ 复制己子：己方活二棋子相邻有空位可形成活三 → 使用。否则不用。
▶ 回溯落子：上步是自己的，且算法显示该位置与对手当前威胁直接相关 → 使用。否则不用。
▶ 连打两子：≥10回合，存在两个空位同时落下可形成必胜定式 → 使用。否则不用。
▶ 强制跳过：≥8回合，需要连续2手且不能被对手中断 → 使用。否则不用。

棋盘: 15x15, x=列 y=行(0-14), .=空 X=黑 O=白 *=封锁

输出JSON: {"activate_skill": {"name": "activate_skill", "args": {...}} | null, "reasoning": "1-2句话"}

规则:
1. activate_skill为null表示不激活技能
2. args需包含目标坐标（若技能需要选位）
3. reasoning需引用game_report中的具体信息
```

### 5.2 战术官提示词（增量更新）

在现有提示词末尾追加：

【棋盘表示】
棋盘显示使用 X(黑棋,奇数号) / O(白棋,偶数号) / .(空) / *(封锁)。
【落子序列】段落给出每一步的编号和坐标——编号奇偶决定颜色，越大越晚。

### 5.3 防守官提示词（增量更新）

在现有提示词末尾追加：

```
【棋盘表示】
棋盘显示使用 X(黑棋,奇数号) / O(白棋,偶数号) / .(空) / *(封锁)。
落子序列段落给出编号→坐标映射，编号奇偶决定颜色，便于回溯对手的进攻发展过程。

【敌方技能预警】
对手拥有1个主动技能（每局限1次）。防守时须预判对手技能可能造成的威胁：
- 对手若有移除/转化类技能，己方关键连珠棋子可能被破坏
- 对手若有追加/复制/连打类技能，其进攻速度可能突然加快
- game_report中标注了对手技能名称，请根据技能类型调整防守强度
```

## 八、风险与注意事项

1. **int grid 与 Agent可见层的双轨制**：内部存储int（0/1/2/3...），对外通过 `board_to_text()`（X/O/*可视化）和 `board_to_move_sequence()`（编号序列）两种形式注入 `game_report`。Agent同时拥有空间视图和时间视图。`board_codec` 是唯一转换层。
2. **重放功能**：int grid天然支持棋谱重放——按序号递增依次落子即可。
3. **CSV棋谱质量**：94条记录中存在数字跳跃（缺失某些序号），可能因为原始棋谱存在提子/无效着，加载时需清洗。
4. **技能强度平衡**：10技能中"连打两子"（≥10回合）和"强制跳过"（≥8回合）最强，如果测试发现过强可提升可用回合门槛。
5. **技能与落子顺序**：UI需确保技能面板先出现，玩家操作技能后再落子。当前架构已支持（skill_window独立于game_window）。
