# 快速开局通道无限循环 —— 修复报告

**日期**: 2026-05-08  
**版本**: v0.4.2  
**严重程度**: 高（游戏完全卡死）  

---

## 1. 现象描述

人机对战（人类执黑先手）中，人类在中心 (7,7) 落子后，AI 陷入无限循环：

- AI 反复显示"AI 开始思考 → 快速开局通道 → AI 落子完成"
- turn 始终为 1，从不推进
- GUI 无响应，只能强制关闭窗口
- 循环频率约每秒 3-4 次，日志在 40 秒内重复 350+ 条

---

## 2. 日志取证

从 `logs/gomoku_20260508.log` 中提取的关键证据：

```
19:52:20 | 人类落子: (7,7), turn=1, result=None
19:52:21 | AI 开始思考 (turn=1)
19:52:21 | 快速开局通道: turn=1, color=O
19:52:21 | AI 落子完成: Position(x=7, y=7)       ← 与人类同位置!
19:52:21 | AI 开始思考 (turn=1)                  ← turn 未推进!
19:52:21 | 快速开局通道: turn=1, color=O
19:52:21 | AI 落子完成: Position(x=7, y=7)
19:52:21 | AI 开始思考 (turn=1)
...
(重复 350+ 次，直到 19:53:00 GUI 关闭)
```

每次循环仅间隔 ~100ms，无 LLM 调用（turn=1 走硬编码），因此速度极快。

---

## 3. 根因分析

### 3.1 硬编码天元不考虑后手（核心问题）

`orchestrator._quick_opening_move()` 中 turn_count==1 分支始终返回 (7,7)：

```python
if turn_count == 1:
    return FinalDecision(
        move=(7, 7),  # ← 硬编码，不检查棋盘状态
        ...
    )
```

**后果**：人类先手 X 占中心后，AI 后手 O 在 turn_count=1 时也被要求下 (7,7)，该位置已被占。

### 3.2 非法落子导致 turn 不推进

`controller._handle_normal_move()` 中：

```python
if not self.board.is_empty(x, y):
    return None  # ← 直接返回，不递增 turn_count，不切换 turn
```

AI 返回的 (7,7) 已被占 → `handle_click` 返回 None → turn 保持 1 → 仍是 AI 回合。

### 3.3 轮询循环再次触发 AI

`game_window._poll_ai_result()` 末尾：

```python
elif self._is_ai_turn():
    self._root.after(200, self._trigger_ai_move)  # ← 又触发 AI
```

turn 未切换 → 仍是 AI 回合 → 无限递归。

### 3.4 快速开局绕过合法性校验

`analyze()` 中的校验-警告-重试机制仅对完整 LangGraph 路径生效（turn ≥ 5）。快速开局路径 `return self._quick_opening_move(...)` 完全跳过校验。

---

## 4. 修复方案

### 4.1 快速开局落子前校验棋盘（`orchestrator.py`）

新增 `_pick_center_or_nearby(board)` 方法，实现降级选择链：

```
中心 (7,7) 空? ──YES──→ (7,7)
    │NO
    ▼
邻位空? ──YES──→ 第一个空邻位
(6,7)(7,6)(6,6)(8,7)(7,8)(8,8)(6,8)(8,6)
    │NO
    ▼
随机空位 ──→ 任意空位
```

turn_count==1 改为调用此方法，而非硬编码 (7,7)。

### 4.2 LLM 路径也加校验（turn 2-4）

`_quick_opening_move()` 的 turn 2-4 LLM 分支新增：

```python
if not board.is_empty(x, y):
    _log.warning(f"快速开局 LLM 返回非法位置 ({x},{y})，使用邻位兜底")
    x, y = self._pick_center_or_nearby(board)
```

### 4.3 analyze() 双重兜底

`analyze()` 中快速开局返回后新增校验：

```python
if controller.turn_count <= 4:
    decision = self._quick_opening_move(...)
    x, y = decision.move
    if not controller.board.is_empty(x, y):
        # 随机空位兜底
        ...
    return decision
```

即使 `_quick_opening_move` 内部逻辑有遗漏，外层也会拦截。

---

## 5. 影响范围

| 项目 | 说明 |
|------|------|
| 触发条件 | 人类执黑先手 + 占中心，AI 执白后手 |
| 是否影响 AI vs AI | 不影响（AI 先手执黑时中心为空） |
| 是否影响双人对战 | 不影响（不经过 AI 路径） |
| 是否影响 turn ≥ 5 的完整路径 | 不影响（有独立的校验-重试机制） |

---

## 6. 预期效果

- 人机对战（执黑）不再卡死
- AI 后手第一步自动选择中心邻位（如 (6,7) 或 (7,6)），符合开局策略
- 快速开局通道所有非法落子均有兜底，不会导致 turn 停滞
