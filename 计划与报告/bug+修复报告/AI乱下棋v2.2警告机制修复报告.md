# AI "乱下棋" v2.2 —— 警告机制修复报告

**日期**: 2026-05-08  
**版本**: v0.2.3  
**严重程度**: 高  

---

## 1. 现象

v0.2.2 修复棋盘格式后，AI 仍然"乱下棋"。从日志中提取的完整对局数据：

| Turn | AI 决策 | 实际落子 | 说明 |
|------|---------|----------|------|
| 5 | (7,6) | **(14,6)** | 兜底随机→最右列 |
| 6 | (7,5) | **(14,3)** | 兜底随机→最右列 |
| 7 | (7,4) | **(3,7)** | 兜底随机 |
| 8 | (7,3) | (7,3) | 刚好合法 |
| 9 | (7,6) | **(5,7)** | 兜底随机 |
| 10 | (7,5) | **(0,14)** | 兜底随机→最底行 |
| 11 | (7,4) | **(4,3)** | 兜底随机 |
| 12 | (7,3) | **(11,14)** | 兜底随机→右下角 |

**规律**：所有 Agent **始终输出 x=7**（中心列），无论该位置是否已被占据。安全兜底触发后随机选空位 → 大量边角落子 →"乱下棋"。

---

## 2. 根因分析

### 2.1 LLM 模型太弱（核心原因）

战术官/防守官使用 **qwen-turbo**（极轻量模型），无法可靠解析 15×15 文本网格。即使在 v0.2.2 中修复了列号头、添加了图例，qwen-turbo 仍然：

- **无法将文本网格映射为坐标** → 总是输出 (7, y) 作为"安全默认值"
- **给出虚假高置信度**（0.80-1.00）→ 触发共识短路，直接采纳错误建议
- **忽略棋盘实际状态** → confidence 1.00 但位置已占

### 2.2 缺乏反馈闭环

原架构中，AI 输出非法落子后：
1. `trigger_ai_move()` 中的安全兜底静默替换为随机空位
2. AI 完全不知道自己的建议被拒绝了
3. 下一轮用同样的上下文再次输出 x=7 的无效位置
4. 形成"AI 绑死 x=7 + 兜底随机边角"的恶性循环

### 2.3 棋盘信息密度过高

单个 15×15 文本网格有 450+ 字符，即使格式正确，小模型也很难精确解析。缺乏"已占据坐标"的显式列表作为辅助参考。

---

## 3. 修复方案

### 3.1 模型升级

| 角色 | 旧模型 | 新模型 |
|------|--------|--------|
| 战术官 | qwen-turbo | **qwen-plus** |
| 防守官 | qwen-turbo | **qwen-plus** |
| 反对官 | qwen-plus | qwen-plus（不变） |
| 总策划官 | qwen-plus | qwen-plus（不变） |

qwen-plus 具有更强的网格解析和坐标推理能力。

### 3.2 警告-重试机制（核心新功能）

`orchestrator.analyze()` 增加校验循环：

```
game_report ──→ LangGraph invoke ──→ 决策 (x,y)
                    ↑                    │
                    │              board.is_empty(x,y)?
                    │               /              \
                    │             YES               NO
                    │              │                 │
                    │          返回决策        附加警告到 game_report
                    │                            │
                    └──────── 重试(最多3次) ──────┘
                                              ↓
                                    3次均失败 → 随机兜底
```

警告格式：
```
!!! 警告（第1次）: 你上次选的 (7,5) 已被棋子占据(X)。
请仔细看棋盘，只能选 . 空位。禁止选 X/O/* 格。
```

### 3.3 已占据坐标显式列表

`game_report` 新增 `_occupied_section`，在棋盘前显式列出：

```
X已占: (7,7),(7,6),(7,8),(7,5),(8,4)...
O已占: (6,6),(7,9),(8,7)...
封锁*: (7,2)
```

LLM 无需解析网格即可直接知道哪些坐标不可用。

### 3.4 提示词强化

4 个 prompt 均更新：
- 规则#1 明确："必须在 X已占/O已占/封锁* 之外选落子"
- 规则#5 明确："收到 !!!警告 时必须更换坐标，严禁重复上次落子"
- 移除冗余描述，聚焦核心规则

---

## 4. 预期效果

| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| AI 合法落子率 | ~10% | ~95%+ |
| 安全兜底触发率 | ~90% | ~1% |
| 警告重试命中率 | N/A | ~80%（1次警告即纠正） |
| 边角落子比例 | ~80% | ~20% |
| 单步 LLM 调用数 | 2-3 次 | 2-4 次（含1次警告重试） |

## 5. 保留的兜底

即使 3 次警告重试均失败，仍通过随机空位兜底保证游戏不卡死。但经过模型升级 + 提示词强化 + 警告机制后，该兜底几乎不会被触发。
