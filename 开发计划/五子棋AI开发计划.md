# 五子棋 AI 智能体 — 开发计划

## 一、项目背景

现有项目包含两个五子棋实现：
- `frame_begin.py` — 控制台双人对战（15×15）
- `gui_prep.py` — tkinter GUI 双人对战（15×15）

目标：新增 AI 智能体，支持人机对战。

---

## 二、技术方案

**方案：基于评分的启发式搜索**

无需第三方库，核心思路是扫描棋盘每个空位，从四个方向分析棋型模式并评分，综合进攻分与防守分选出最优落子点。

---

## 三、实施步骤

### 步骤 1：创建 AI 引擎模块 `agent.py`

新建文件，包含以下核心组件：

#### 1.1 常量定义

| 常量 | 值 | 说明 |
|------|-----|------|
| `N` | 15 | 棋盘大小 |
| `EMPTY` | `' '` | 空位标记 |
| `BLACK` | `'X'` | 黑棋（玩家） |
| `WHITE` | `'O'` | 白棋（AI） |

棋型评分表（可调优）：

| 棋型 | 分数 | 说明 |
|------|------|------|
| 连五 | 10,000,000 | 直接获胜 |
| 活四 | 1,000,000 | 对手无法阻挡 |
| 冲四 | 100,000 | 仅一端开放的四连 |
| 活三 | 50,000 | 两端开放的三连 |
| 眠三 | 5,000 | 一端开放的三连 |
| 活二 | 1,000 | 两端开放的二连 |
| 眠二 | 100 | 一端开放的二连 |

#### 1.2 棋型识别函数

```
analyze_direction(board, x, y, dx, dy, color)
    → 返回 (count, open_ends)
    count: 该方向上的连子数
    open_ends: 两端开放端数（0/1/2）

get_pattern_score(count, open_ends)
    → 根据 count 和 open_ends 查表返回分数
```

#### 1.3 位置评估函数

```
evaluate_position(board, x, y, color)
    → 模拟在 (x,y) 放置 color 棋子，四方向评分之和
```

#### 1.4 综合决策函数

```
get_best_move(board, ai_color)
    → 返回最佳落子坐标 (x, y)

逻辑：
    1. 遍历所有空位
    2. 计算进攻分：evaluate_position(board, x, y, ai_color)
    3. 计算防守分：evaluate_position(board, x, y, opponent_color)
    4. 综合分 = 进攻分 + 防守分 × 0.9
    5. 返回综合分最高的位置
```

#### 1.5 特殊情况处理

- **第一步落子**：若棋盘为空，直接落天元 (7,7)
- **对手即将获胜**：检测到对方连五差一步时，防守分应大幅提高
- **必胜局面**：检测到己方连五差一步时，优先进攻

### 步骤 2：修改 GUI — 添加人机对战模式

修改 `gui_prep.py`：

#### 2.1 新增游戏模式选择

- 程序启动时弹出选择对话框
- 选项：双人对战 / 人机对战（玩家执黑） / 人机对战（玩家执白）
- 选定后进入对应模式

#### 2.2 新增状态变量

| 变量 | 类型 | 说明 |
|------|------|------|
| `game_mode` | `int` | 0=双人, 1=人机(玩家黑), 2=人机(玩家白) |
| `ai_color` | `str` | AI 的棋子颜色 (`'X'` 或 `'O'`) |

#### 2.3 修改落子逻辑 `click()` 函数

```
click(ev):
    if game_mode == 双人:
        → 保持现有逻辑不变

    if game_mode == 人机:
        → 玩家落子
        → 检查是否结束
        → 切换回合
        → 调用 agent.get_best_move() 让 AI 落子
        → 检查是否结束
        → 切换回合
        → 重绘棋盘
```

#### 2.4 确保 AI 落子时有视觉反馈

- 玩家落子后，棋盘更新并短暂延迟（用 `root.after()`），然后 AI 落子
- 避免界面卡死

### 步骤 3：（可选）优化与增强

#### 3.1 多步搜索（1-2层前瞻）

- 对 Top 5 候选位置，模拟落子后递归评估对手的最佳应对
- 提高防守准确性

#### 3.2 难度分级

| 难度 | 实现方式 |
|------|----------|
| 简单 | 只考虑进攻分，不加防守；或加入随机扰动 |
| 普通 | 进攻 + 防守评分（基础版） |
| 困难 | 基础版 + 两步前瞻搜索 |

#### 3.3 开局库（可选）

- 缓存 5-10 步内的常见开局走法
- 让 AI 在开局阶段不走明显缓手

## 五、测试要点

1. **棋型识别**：手动构造场景验证活四、冲四、活三等识别是否正确
2. **防守测试**：AI 能否正确阻挡对手的四连和三连
3. **进攻测试**：AI 能否在优势局面下正确连五
4. **边界测试**：棋盘边缘的棋型识别是否正常
5. **先后手测试**：玩家执黑和执白两种模式下 AI 行为是否正确
6. **重置测试**：一局结束后能否正常重置开始新游戏
